A bar series represents data points as **boxes** that span the distance between the data point value and the zero value in along one axis, and an arbitrary, stylistic distance along the other axis. It is often used when one axis is categorical. When multiple bar series are shown, the bars are offset along this second axis such that they are shown side-by-side.

```javascript react-live=true
<Chart height={100} view={[0, 0, 4, 5]} isCanvas={false}>
  <XAxis />
  <YAxis />
  <BarVerticalSeries
    data={[
      [1, 3],
      [2, 2],
      [3, 5],
    ]}
    chartStyle={{
      dataBoxFill: 'blue',
      seriesXOffset: -5,
    }}
  />
  <BarVerticalSeries
    data={[
      [1, 1],
      [2, 1.5],
      [3, 2.5],
    ]}
    chartStyle={{
      dataBoxFill: 'red',
      seriesXOffset: 5,
    }}
  />
</Chart>
```

The [`seriesStyles` helper functions](/basic-charts/7-series-styles/) can be used to generate `dataBoxLeftOffset` values.

Note that the current name of the Bar series component is `BarVerticalSeries`.

## Props

import PropDef from '../components/PropDef';

<PropDef name="data" type="Point[]" required>
  An array of Points (itself a bipartite array of [x, y] values).
</PropDef>
<PropDef name="view" type="[number, number, number, number]">
  A viewbox, specified as [xMin, yMin, width, height]. Any data point that falls
  outside of this box will not be rendered. The default value is the viewbox of
  the Chart.
</PropDef>
<PropDef name="chartStyle">
  Styles declarations which will override the global{' '}
  <a href="/basic-charts/2-chart-style#databoxes">chart style</a>. Usually, the
  databox styles are used with this type of data series.
</PropDef>
<PropDef name="handlerMeta" type="Record<any>">
  Metadata to pass to{' '}
  <a href="/interaction/pointer-events/">handler callbacks</a>.
</PropDef>
<PropDef name="view" type="[number, number, number, number]">
  A viewbox, specified as [xMin, yMin, width, height]. Any part of the data
  which falls outside of this viewbox will be clipped (not shown). The default
  value is the viewbox of the Chart. If `null` is passed explicitly, no clipping
  will occur (data may render in the gutter).
</PropDef>
<PropDef
  name="renderBar"
  type="React.Component<{x: number; yMin: number; yMax: number; chartStyle }>"
>
  Render prop for the data boxes rendered by this series type. See the{' '}
  <a href="/composing-charts/1-data-components/">data component docs</a> for
  more information. By default it renders a `DataRangeVertical` and a
  replacement component should accept the same props.
</PropDef>

Additionally, the bar series component accepts all the [pointer event handlers](/interaction/1-pointer-events/), which are rendered around each individual bar.
